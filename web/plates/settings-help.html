The Settings page provides both control and configuration of your Stratux device.

Use the toggles in the Hardware section to control which devices are active.

NOTE: Only hardware toggled on here will appear on the Status page.

WiFi

Stratux can operate in several WiFi modes:

  • Access Point only (default) — Stratux creates its own WiFi network. Your EFB connects directly to it.
  • Access Point + Client — Stratux creates its own network and connects to an existing network (e.g. a phone hotspot). Use this to enable internet connectivity for OGN APRS position reporting.
  • Client only — Stratux connects to an existing network and does not create its own. Advanced use only.

WARNING: Misconfiguring WiFi settings can make your Stratux unreachable via the web interface. Recovery requires either SSH access over a wired connection or reflashing the SD card. Change these settings carefully.

WiFi SSID
The name of your Stratux network. Consider setting this to your aircraft tail number (e.g. Stratux-N12345) so it is easy to identify in the EFB connection list.
Network Security
Turns wireless password protection On or Off. Disable only if your EFB has difficulty connecting.
WiFi Passphrase
Password for your Stratux network when security is enabled.
WiFi Channel
Changing the channel can help in congested WiFi environments (e.g. busy airports or FBOs).
WiFi Client Network
SSID and password of an existing network to connect to (phone hotspot, home router, etc). Used for internet-dependent features such as OGN APRS.

Diagnostics

  • Toggling Traffic Source adds a two-letter prefix to callsigns in your navigation app, indicating where each target was received. For example: ea = 1090 MHz air-to-air ADS-B, ur = 978 MHz ADS-R rebroadcast, fl = FLARM, fn = FLARM network / PilotAware. Useful for understanding what your Stratux is hearing.
  • Toggling Record Logs enables logging to a series of files for your Stratux device including data recorded for UAT traffic and weather, 1090 traffic, GPS messages, and AHRS messages. Log files are accessible from the Logs menu on the left. NOTE: Logs are stored in RAM by default and are lost on reboot. Enable Persistent Logging (below) to write logs to the SD card instead. Persistent logging increases SD card wear — use it for troubleshooting, not permanently.
  • Persistent Logging — Writes logs to the SD card rather than RAM so they survive a reboot. A reboot is required after enabling this setting. Increases SD card write cycles. Recommended for troubleshooting sessions only.

AHRS

Calibrate AHRS Sensors guides initial setup of the AHRS function, specifying the orientation of the sensors relative to the airplane. You only need to do this once. The direction of gravity is used to determine which axis is forward.

To find your Minimum fan duty cycle %: start at 50% and wait for the fan to spin up. Decrease slowly until the fan runs smoothly at the lowest stable RPM without stalling.

GLoad Limits sets the values shown on the G Meter on the GPS/AHRS page. Enter a space-separated list, e.g. -1.76 4.4.

Configuration

  • Ownship Mode S / OGN Code — Enter your aircraft's ICAO 24-bit code (hexadecimal, not octal) to prevent your own transponder from appearing as traffic. You can find this value on popular flight tracking websites by searching your tail number. Set the address type to ICAO after entering the code. Stratux will automatically determine which code is flying with you if you enter multiple codes for aircraft you fly regularly.
  • Weather Watch List — A space-separated list of airport or navaid identifiers (e.g. KBOS EEN LAH). Stations on this list appear at the top of the Weather page. Weather data for all stations within range is always received regardless of this list — it affects display order only. NOTE: Changes take effect immediately but must be confirmed by tapping elsewhere or pressing ENTER.
  • PPM Correction — A frequency offset correction value for your SDR dongle. Most modern RTL-SDR dongles have low enough error that a value of 0 works well. If you suspect PPM error (e.g. consistently missing traffic that others report), try adjusting in small increments (±5 PPM) and test in flight.
  • GDL90 bearingless target circle emulation — Stratux can receive altitude reports from Mode C/S transponders that do not transmit ADS-B position. When using NMEA output, these appear as circles in most EFBs. GDL90 does not natively support this; enabling this option creates 8 pseudo-targets in a ring to emulate the display in GDL90-based apps.

Settings Storage and Migration

All settings are stored in stratux.conf on the boot partition of the SD card (/boot/firmware/stratux.conf). This partition uses the FAT32 format and is directly readable and writable when you plug the SD card into any computer — no special software required.

To back up or migrate settings between SD cards or Stratux units:

  1. Shut down Stratux and remove the SD card.
  2. Insert the SD card into a computer. The boot partition will mount automatically on most systems.
  3. Copy stratux.conf to your computer (backup) or copy a saved stratux.conf onto the new card (restore/migrate).
  4. Reinsert the card and boot Stratux. Your settings will be applied immediately.

NOTE: After a major version upgrade (e.g. v1.x to v2.x) some settings fields may have changed. If Stratux behaves unexpectedly after migration, try resetting to defaults and reconfiguring.

System

  • Shutdown — Safely shuts down the Stratux. Wait for the status light to go out before removing power.
  • Reboot — Immediately reboots the Stratux. You may need to rejoin the WiFi network after reboot.
